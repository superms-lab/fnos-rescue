# 第四阶段：Web 引导式恢复闭环

状态：已完成（2026-07-12）

## 本阶段交付

- 在 Web 控制台中完成源盘保护、案例创建、目标目录验证、复制任务创建和后台启动的连续流程。
- 增加任务暂停、继续与取消控制；继续操作会重新启动可续作 worker。
- 案例统一存放于 `~/.local/share/fnos-rescue/cases`，案例 ID 和任务 ID 均严格校验，不能用于目录穿越。
- 案例创建前再次验证设备序列号，并强制要求源盘已处于只读状态。
- 目标目录必须通过可写性、剩余容量和物理非同盘检查。
- Web 任务类型采用固定白名单，仅开放 `copy`、`verify` 和 `btrfs-probe`，不接受 shell 或任意命令。
- 复制任务继续复用核心层的相对路径约束、符号链接拒绝、临时文件落盘和逐文件 SHA-256 校验。

## 安全说明

macOS 与其他非 Linux 平台不显示任何伪造设备，只显示空设备状态。真实操作按钮只会在 Linux/fnOS 返回真实块设备后开放；本机服务不可达时会明确显示连接错误。

Web 服务默认仅监听 `127.0.0.1:8790`。局域网监听必须由管理员显式指定，并配合可信网络和防火墙。
