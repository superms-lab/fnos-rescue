# 第三阶段：共用本地 Web UI

状态：已完成（2026-07-12）

## 本阶段交付

- React + Vite 恢复控制台，适配桌面与窄屏浏览器。
- 本机只读设备扫描 API；非 Linux 环境自动进入清晰标注的安全演示模式。
- `fnos-rescue serve` 命令，默认仅监听 `127.0.0.1:8790`。
- 源盘只读保护入口，必须手工输入设备完整序列号，复用核心层的串号校验与逐层只读验证。
- 拒绝跨源写请求、非 `/dev/` 路径和超大请求体；API 响应包含 CSP、禁止嗅探及禁缓存头。
- fnOS 原生包可携带生产构建后的 Web 静态资源。

## 启动

```bash
cd web && npm install && npm run build
fnos-rescue serve
```

浏览器打开 `http://127.0.0.1:8790`。如需从局域网访问，管理员必须显式传入 `--host`，并自行配置可信防火墙；工具不会默认暴露到网络。

## 当前安全边界

Web UI 当前只开放设备枚举与源盘只读保护。创建案例、选择目标和启动恢复仍保持禁用，直到后续阶段为这些流程补齐逐步确认、权限隔离和端到端验收。界面中的活动任务及故障摘要标有“界面预览数据”，不会伪装成真实恢复结果。
